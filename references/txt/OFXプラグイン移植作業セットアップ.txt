Autodesk Flame 2025におけるOpenFXプ
ラグイン移植：詳細セットアップガイド
Autodesk Flame 2025（Rocky Linux 8.10 / DKU 20.0.0）環境における、OpenFX（OFX）プラグインの
ビルドおよびデプロイメントのための詳細な開発セットアップ手順を記述する。本ガイドでは、開発リ
ソースを高速なRAIDボリュームに配置し、ビルドされたバイナリをシステム標準のプラグインディレク
トリへ展開するワークフローに最適化している。


1. 開発環境のディレクトリ構成
移植作業のベースとなるソースコードおよびビルド環境は以下の通り設定する。



 項目                        パス / 設定値                     備考


 開発ルート (Git repo)          /Volumes/RAID/develop/ofx/   ユーザー指定の開発ベース
                           openfx                       パス


 SDKインクルードパス               /Volumes/RAID/develop/ofx/   OFX標準ヘッダー群 1
                           openfx/include


 C++ Support Lib           /Volumes/RAID/develop/ofx/   移植に推奨されるC++ラッ
                           openfx/Support               パー 1

 標準インストール先                 /usr/OFX/Plugins             システム標準の検索パス 4


2. システム基盤とツールチェーンの有効化
Rocky Linux 8.10およびDKU 20.0.0は、VFX Reference Platform CY2024の要件（glibc 2.28等）を
満たしている 。Flame 2025とのバイナリ互換性を確保するため、GCC Toolset 13以降を使用する。

ツールチェーンのインストールと起動
開発用シェルを起動するたびに、以下のコマンドを実行して最新のコンパイラ環境を有効化する 。



  Bash
# GCC Toolset 13のインストール（未導入の場合）​
sudo dnf install gcc-toolset-13​
​
# 開発セッションでの有効化​
scl enable gcc-toolset-13 bash​

必須依存パッケージ (DNF)
ビルドおよびOpenGLインターフェースに必要なライブラリを導入する 。



  Bash



sudo dnf install mesa-libGL-devel libX11-devel expat-devel​


3. ビルド構成（CMake）の設定
/Volumes/RAID/develop/ofx/openfx 内でCMakeを使用し、Flame 2025に適合したビルドバイナリを
生成する。

CMakeLists.txt の重要設定
共有ライブラリの生成において、以下のフラグを維持・追加する 。
 ●​ CMAKE_CXX_STANDARD: 17 以上を推奨 。
 ●​ POSITION_INDEPENDENT_CODE: 共有ライブラリとしてロードするため ON に設定。
 ●​ Visibility: デフォルトのシンボル公開設定を hidden とし、OFXエントリポイントのみを default
    で公開する 。

ビルド実行例


  Bash



cd /Volumes/RAID/develop/ofx/openfx​
mkdir build && cd build​
cmake -DCMAKE_BUILD_TYPE=Release..​
make -j$(nproc)​
4. Flame 2025 へのデプロイメント
ビルドされた .ofx バイナリは、OpenFX規格で定められたバンドル構造（Directory Hierarchy）に従っ
てインストール先に配置する必要がある 4。

バンドル構造の構築
システム標準のディレクトリに以下の構造で展開する。

/usr/OFX/Plugins/

└── OpenDefocus.ofx.bundle/

└── Contents/

└── Linux-x86-64/

└── OpenDefocus.ofx (ビルドされたバイナリ)

インストールコマンド例


  Bash



sudo mkdir -p /usr/OFX/Plugins/OpenDefocus.ofx.bundle/Contents/Linux-x86-64​
sudo cp OpenDefocus.ofx /usr/OFX/Plugins/OpenDefocus.ofx.bundle/Contents/Linux-x86-64/​


5. Flame固有の実行時設定とデバッグ
Flame 2025は、起動時に /usr/OFX/Plugins を自動的にスキャンする 。開発中の変更を即座に反映
させるため、キャッシュ制御およびデバッグログの有効化が推奨される。

プラグイン・キャッシュの管理
Flameはプラグイン情報を /opt/Autodesk/flame_2025/status/ofxPluginCache.xml に保持する 9。
kOfxActionDescribe などの記述情報を変更した場合は、以下のコマンドでキャッシュを削除し、再
起動時に強制スキャンさせる必要がある。



  Bash
# キャッシュの削除​
rm /opt/Autodesk/flame_2025/status/ofxPluginCache.xml​

また、環境変数 DL_OFX_PLUGIN_CACHE_ENABLED を 0 に設定することで、開発期間中のキャッ
シュ機能を一時的に無効化できる 9。

デバッグ環境変数の活用
Flameのシェルログに詳細なOFX通信ログを出力させ、DescribeやRenderアクションの成否を確認
する 9。



  Bash



export OFX_DEBUG_ALL=1​
export OFX_DEBUG_ACTIONS=1​


6. 画像演算におけるメモリ・アドレッシング
Flame 2025から渡されるピクセルデータへのアクセスには、Rocky Linux環境でのメモリアライメント
を考慮した以下の計算モデルを使用する 2。




 ●​ rowBytes: ホストから提供される行あたりの物理的なバイト幅（パディングを含む）。
 ●​ pixelBytes: ピクセル深度（32-bit Floatの場合はチャンネル数×4バイト） 。


7. セットアップ完了の検証
 1.​ OS/DKUの確認: cat /etc/rocky-release が 8.10 を示し、/opt/Autodesk/DKU/current が
     20.0.0 であること 。
 2.​ UEFIの確認: /sys/firmware/efi が存在すること（Rocky 8.10以降、UEFIはFlame 2025の必須要
     件） 。
 3.​ バイナリスキャンの確認: Flame起動後、Batchノードの「OpenFX」カテゴリに自作プラグインが
     表示されることを確認する 。

本構成により、高速なRAIDボリュームでの効率的なコンパイルと、Flame 2025標準環境へのスムー
ズなデプロイメントが両立可能となる。
